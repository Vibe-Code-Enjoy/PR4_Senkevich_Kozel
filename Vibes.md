# Отчет для автоматизированной системы музея искусства создания мемов и брейнрота

## 1. Введение

**Цель работы:**
Основной целью данной работы является проведение полномасштабного системного тестирования веб-приложения «Мемотека». Тестирование направлено на проверку работоспособности всех интерактивных элементов (лайки, удаление, смена статуса), верификацию корректного отображения медиа-контента из внешних источников и подтверждение того, что программный продукт полностью соответствует ожиданиям пользователя в плане хранения и фильтрации данных.

**Краткое описание программы:**
Веб-сайт представляет собой динамическую платформу для каталогизации и управления коллекцией мемов. Ресурс позволяет пользователям добавлять новые записи через форму, взаимодействовать с контентом посредством системы лайков и управлять видимостью карточек через переключение статусов («Активен» / «Неактивен»). В качестве бэкенда используется PHP, а хранение данных реализовано на базе локального JSON-файла, что обеспечивает быструю синхронизацию состояния приложения без использования внешних систем управления базами данных.

**Обоснование тестирования:**
Необходимость системного тестирования обусловлена архитектурными особенностями приложения, где корректность обработки данных напрямую зависит от стабильности записи в файловую систему и безопасности обработки пользовательского ввода. В ходе проверки оценивается целостность структуры JSON-хранилища после манипуляций с контентом, правильность работы алгоритмов фильтрации и устойчивость интерфейса к попыткам внедрения вредоносного кода через поля ввода.

## 2. Тест-план (Test Plan)

### 2.1 Цели и задачи
* Верификация корректности выполнения CRUD-операций.
* Проверка логики переключения состояний объектов («Активен» / «Неактивен»).
* Подтверждение работоспособности системы фильтрации контента в зависимости от выбранного статуса.
* Проверка корректности записи данных в файл `memes.json` и их последующего считывания.
* Проверка безопасности приложения на устойчивость к XSS-инъекциям через формы ввода.
* Контроль сохранения состояния фильтра при выполнении POST-запросов и редиректов.

### 2.2 Область применения
**Тестируемые модули:**
* **Модуль управления данными:** форма добавления мемов и обработка входящих POST-запросов.
* **Интерфейс карточек:** интерактивные кнопки «Лайк», «Сменить статус» и «Удалить».
* **Система фильтрации:** навигационная панель переключения режимов просмотра («Все», «Активные», «Неактивные»).
* **Хранилище данных:** структура и целостность файла `data/memes.json`.
* **Валидация вывода:** механизмы экранирования спецсимволов в заголовках.

**Ограничения:**
* Тестирование производительности при экстремально большом количестве записей в JSON не проводится.
* Тестирование сетевых задержек и кроссдоменных политик при загрузке изображений ограничено проверкой атрибута `onerror`.

### 2.3 Стратегия тестирования
* **Функциональное тестирование:** проверка базовой логики приложения (лайки, удаление, добавление).
* **Тестирование логики:** проверка алгоритмов фильтрации и переключения статусов.
* **Тестирование безопасности:** проведение проверок на внедрение вредоносного кода (Injection testing).
* **Интеграционное тестирование:** проверка взаимодействия PHP-скрипта с файловой системой (чтение/запись JSON).
* **GUI-тестирование:** визуальный контроль отображения карточек и меток статуса.
* **Регрессионное тестирование:** проверка корректности работы исправленных модулей после выявления багов.

В данном разделе представлены сгруппированные тест-кейсы, направленные на проверку ключевых функциональных областей приложения «Мемотека».

### 3.1 Функциональное тестирование
Данный набор тестов проверяет работоспособность стандартных сценариев использования приложения.

* **TC-01:** Добавление мема с корректными данными.
* **TC-02:** Увеличение счётчика лайков при нажатии на кнопку "Лайк".
* **TC-05:** Удаление существующего мема из системы.

### 3.2 Тестирование бизнес-логики и фильтрации
Набор тестов для проверки алгоритмов обработки состояний объектов и корректности выборки данных.

* **TC-03:** Смена статуса мема с "Активен" на "Неактивен" и проверка автоматического обновления.
* **TC-04:** Отображение активных или неактивных мемов через панель фильтров.

### 3.3 Тестирование безопасности и целостности данных

* **Сквозная проверка JSON:** Верификация структуры файла `memes.json` после выполнения каждой операции из списка выше.
